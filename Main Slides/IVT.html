Migration Program - Test Phase Plan

📊
Test phase When Who Dependencies
IVT: Infrastructure Verification Testing After each stage of new infrastructure build in target is complete Kyndryl build team Build is complete
SMT: Server Migration Testing In source and target for each event Kyndryl migration team Build complete, IVT complete, and other migration entry criteria met
TVT: Technical Verification Testing In source once (for timings), in target at each event, and twice at cutover Kyndryl migration team / IMI support team IMI/Migration architect to provide tests per interface and other points of change. SMT complete and test entry criteria met
AVT: Application Verification Testing In source once (for timings), in target at each event, and twice at cutover Apps support teams Apps support teams provide tests per application. Normally TVT is fully complete
BVT: Business Verification Testing In source once (for timings), in target at each event, including at cutover (non-prod apps) Business teams Business provides tests per application. AVT associated with an application must complete before BVT starts (phase overlap may be possible). AVT must be complete
UAT: User Acceptance Testing In source once (for timings), in target at each event, including at cutover (production apps) Business teams Business provides tests per application. AVT associated with an application must complete before UAT starts (phase overlap may be possible). AVT must be complete
Field Of View Only after post-cutover BVT passed and test exit agreed. May be on cutover day or days following Business teams plus any app support post-processing Exit from cutover testing has been agreed
Planning principle: complete each testing dependency gate before starting the next phase in the migration wave.
